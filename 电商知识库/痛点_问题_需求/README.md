# 痛点收集 · 方法说明

## 为什么不按平台分类？

电商领域太大，我们的 AI Agent 不可能覆盖全平台所有问题。

**真正值得做的方向 = 跨平台都存在的通病**

按平台收集痛点会让我们陷入细节，最后一堆碎片难以决策。
正确的做法是：**按卖家的业务职能分类**，找出哪些环节在所有平台都痛。

## 核心逻辑

```
不管你做亚马逊、TikTok、Temu 还是独立站
你每天都要做这几件事：

  选品  →  上架  →  引流  →  客服  →  发货  →  算账

某个环节在所有平台都是痛点 → 这就是我们要做的 AI Agent 方向
```

## 参与方式

1. 进入 `职能痛点矩阵.md`，找到你最熟悉的职能模块
2. 在对应行填写你的痛点，标注你做的平台
3. 如果觉得某个痛点特别严重，在后面加 `🔥`
4. 不确定放哪就写在文件末尾的「待归类」区域

## 文件说明

| 文件 | 用途 |
|------|------|
| `职能痛点矩阵.md` | **主文档**，按业务职能组织，跨平台对比 |
| `平台特有痛点.md` | 记录某平台独有的规则/机制类痛点（补充用） |

---

> 平台差异是皮，业务职能是骨。抓住骨头，才能做出通用的 AI Agent。
