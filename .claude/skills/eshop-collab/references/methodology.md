# 项目方法论完整版

## 四步流程

```
第一步          第二步            第三步          第四步
痛点收集  →  评分 & 筛选  →  Agent 定义  →  Vibe Coding
```

## 第二步：痛点评分公式

优先级得分 = 严重程度(1-5) × 频次(1-5) × AI可行性(1-5) × 市场规模(1-5)

- ≥ 300 分 → 进入候选方向
- ≥ 200 分且无好工具 → 进入候选方向
- 付费意愿 ≥ $50/月 → 商业价值够

## 第三步：Agent 定义文档结构

在 `方向/` 目录下创建，包含：
- 解决的核心问题（一句话）
- 目标用户画像
- 做什么 / 不做什么（功能边界）
- 输入 & 输出
- 成功标准（可量化）
- 竞品分析
- MVP 最简版本

## 第四步：开发节奏

每轮 1 周：周一定目标 → 周三检查 → 周五演示 → 周末整理
每个任务一人认领，两天内完成
